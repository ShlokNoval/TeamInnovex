2. Two Analysis Modes Overview  
2.1 Mode A — Real -Time CCTV Analysis  
In real -time mode, the system continuously reads frames from a live CCTV stream (RTSP feed or 
equivalent). Each frame is processed through the YOLO detection pipeline, annotated with bounding 
boxes and labels, and the result is streamed back to the monitor ing dashboard. Incidents are raised 
when confirmed hazard conditions are met across multiple consecutive frames.  
 
Flow: RTSP Stream → OpenCV Frame Capture → YOLOv8 Detection → Annotation → Severity 
Scoring → Incident Creation → WebSocket Push → Dashboard Display

IoU is the fundamental metric used by YOLO to evaluate the quality of bounding box predictions 
against ground truth. It is also used in NMS and tracking.  
 
IoU(A, B) = Area(A ∩ B) / Area(A ∪ B) 
 
Where A is the predicted bounding box and B is the ground truth box. IoU ranges from 0 (no overlap) to 
1 (perfect overlap). A detection is considered valid when IoU ≥ threshold (typically 0.45 for NMS, 0.5 for 
evaluation).  
 
3.3 YOLO Confidence Score  
Each detected object is assigned a confidence score combining objectness probability and class 
probability:  
Confidence = P(Object) × P(Class | Object) = P_obj × P_cls  
 
Filter Rule: Detections with Confidence < 0.45 are discarded. Hazard incidents require Confidence 
>= 0.55. High -severity alerts require Confidence >= 0.70.  
 
3.4 Non -Maximum Suppression (NMS)  
NMS prevents duplicate detections of the same object by suppressing lower -confidence overlapping 
boxes:  
Keep box B_i if: Confidence(B_i) > Confidence(B_j) for all j where IoU(B_i, 
B_j) >= NMS_threshold  
 
NMS threshold is set to 0.45. This is critical for road scenes with dense vehicle clusters and 
overlapping pothole regions.  
 
3.5 Normalized Bounding Box Coordinates  
YOLO outputs normalized coordinates relative to image dimensions. These must be converted for 
pixel-space operations:  
x_pixel = x_norm × img_width  
y_pixel = y_norm × img_height  
w_pixel = w_norm × img_width  
h_pixel = h_norm × img_height  
Bounding Box Area = w_pixel × h_pixel

4.1 Pothole Severity Score  
Pothole severity is computed as a weighted combination of bounding box area (normalized), detection 
confidence, road zone position, and repeated observation frequency.  
 
Pothole Severity Score Formula  
 
SS_pothole = w1 × A_norm + w2 × C + w3 × Z + w4 × F_repeat  
 
Variable  Definition  Formula / Value  
SS_pothole  Pothole Severity Score (0 –100) Composite weighted score  
A_norm  Normalized bounding box area  A_norm = (w_px × h_px) / (img_w × img_h)  
C YOLO detection confidence  Output from YOLOv8, range [0,1]  
Z Road zone position weight  Centre lane = 1.0, edge = 0.5, offroad = 0.2  
F_repeat  Repeat detection frequency  F = detections_in_last_10_frames / 10  
w1 Area weight  0.35 
w2 Confidence weight  0.30 
w3 Zone position weight  0.20 
w4 Frequency weight  0.15 
 
Scale Rule: SS_pothole × 100 maps to 0 –100. Low: 0 –33, Medium: 34 –66, High: 67 –100. 
 
Road Zone Classification  
Zone = 'CENTRE' if (img_width/4 < x_centre < 3×img_width/4) else 'EDGE' or 
'OFF_ROAD'  
 
4.2 Pothole Area Growth Rate (Before/After Tracking)  
For recurring pothole detections, the system tracks the area change across time to determine whether 
the pothole is worsening, stable, or resolved.  
Growth_Rate = (A_after - A_before) / A_before × 100  (percentage change)  
Status = 'GROWING'   if Growth_Rate > +15%  
Status = 'STABLE'    if -15% <= Growth_Rate <= +15%  
Status = 'RESOLVED'  if Growth_Rate < -80% (pothole no longer detected)

5. Motion Analysis Formulas  
5.1 Centroid Tracking  
Object tracking across frames uses centroid -based matching. The centroid is computed from bounding 
box coordinates:  
Centroid = ((x + w/2), (y + h/2))  
Distance(C1, C2) = sqrt((x2 -x1)^2 + (y2 -y1)^2)   [Euclidean]  
Objects within a distance threshold (D_max = 50 pixels) between frames are matched as the same 
tracked object. If no match is found within D_max, the object is treated as new.  
 
5.2 Optical Flow — Motion Anomaly Score  
The motion anomaly score uses dense optical flow (Lucas -Kanade or Farneback) to detect sudden 
unusual motion patterns indicative of accidents:  
Flow_magnitude(x,y) = sqrt(u(x,y)^2 + v(x,y)^2)  
AI Road Hazard Detection — Analysis Engine Report  | Confidential  
Page 8 | PS4: AI -Based Road Hazard Detection  Mean_flow = (1/N) × Σ Flow_magnitude(x,y)  for all N pixels in ROI  
σ_flow = sqrt((1/N) × Σ (Flow_magnitude - Mean_flow)^2)  [Standard 
deviation]  
M_anomaly = min(1.0, σ_flow / σ_baseline)  
 
σ_baseline: The baseline standard deviation of flow is computed as a rolling average over 30 normal 
frames at session start. Sudden spikes (σ_flow >> σ_baseline) indicate collision or abrupt motion.  
 
5.3 Pixel Displacement Velocity  
To estimate object velocity in pixel -space (used for both animal behavior and accident speed proxy):  
V_pixel(t) = Distance(Centroid(t), Centroid(t -1)) / frame_interval  
V_pixel_avg = (1/n) × Σ V_pixel(t)  for last n frames (n=5 default)  
V_kmh_approx = V_pixel × (known_road_width_m / road_width_pixels) × FPS × 
3.6 
 
5.4 Sudden Stop Detection  
Hit-and-run and accident detection relies on identifying vehicles that suddenly decelerate:  
Deceleration(t) = V_pixel(t -1) - V_pixel(t)  
Sudden_Stop = True  if Deceleration(t) > V_pixel(t -1) × 0.7  (70% velocity 
drop in 1 frame)  
Hit_and_Run_Flag = True  if Sudden_Stop AND vehicle exits frame within 5 
frames  
 
5.5 Traffic Density and Congestion Level  
Congestion level is computed as a normalized vehicle density relative to the road capacity estimate:  
Traffic_density = N_vehicles_in_ROI / ROI_area_pixels  
Congestion_Level = min(1.0, Traffic_density / Traffic_density_max)  
Congestion_Label: LOW (<0.33), MEDIUM (0.33 –0.66), HIGH (>0.66)  
 
6. Priority Index — Unified Ranking Formula  
6.1 Incident Priority Index (PI)  
The Priority Index is a single number that ranks all active incidents from all hazard types to help 
authorities triage response actions. It combines severity, confidence, recency, and hazard category 
weight.

PI = w_type × SS_normalized × C × T_decay  
 
Variable  Definition  Value / Formula  
PI Priority Index (0 –100) Output ranking score  
w_type  Hazard type weight  Accident = 1.0, Animal = 0.8, Pothole = 0.6  
SS_normalized  Severity score normalized to [0,1]  SS / 100  
C Detection confidence  YOLOv8 output [0,1]  
T_decay  Time decay factor  T_decay = exp( -λ × Δt)  where Δt = seconds 
since detection  
λ (lambda)  Decay rate  0.001 (slow decay over 15 min to near 0.4)  
 
Usage: All open incidents are sorted by PI descending. The top -K incidents are surfaced in the 
priority alert panel on the authority dashboard. PI is recalculated every 30 seconds.  
 
T_decay = e^( -0.001 × Δt)   [After 15 min: decay ≈ 0.41; after 60 min: decay 
≈ 0.05]

7. Night -Time Detection Pipeline  
7.1 Low -Light Preprocessing Formulas  
Before YOLO inference on night -time frames, preprocessing enhances visibility using histogram 
equalization and contrast adjustment:  
 
CLAHE (Contrast Limited Adaptive Histogram Equalization)  
AI Road Hazard Detection — Analysis Engine Report  | Confidential  
Page 10 | PS4: AI -Based Road Hazard Detection  L_enhanced = CLAHE(L_channel, clip_limit=2.0, tile_grid=(8,8))  
Applied only to the L (luminance) channel after converting BGR to LAB color space. This preserves 
color while boosting dark regions.  
 
Gamma Correction  
I_out = I_in^(1/γ)  where γ < 1 brightens dark images  
Recommended: γ = 0.5 for night scenes, γ = 0.7 for overcast/dusk  
 
Noise Reduction  
I_denoised = GaussianBlur(I, kernel=(3,3), σ=0)  OR  fastNlMeansDenoising(I, 
h=10)  
 
7.2 Adaptive Confidence Threshold  
Night -time inference uses lower confidence thresholds to compensate for reduced feature visibility:  
C_threshold_night = C_threshold_day × 0.80  
For example: Day threshold 0.55 → Night threshold 0.44. This reduces missed detections at the cost of 
marginally more false positives, which are filtered by temporal confirmation.
