AI-Powered Road Hazard Detection 
Full Development Brief and Technical Documentation 
Version 1.0 | Free-only stack | Web-based implementation 
Project Goal 
Detect potholes, stray animals, and accidents from 
CCTV video, generate structured incident reports, 
calculate severity and priority, and notify authorities 
through a dashboard and email alerts. 
 
 
 
1. Project Overview 
Problem statement: Use existing CCTV feeds to detect road hazards in real time and report them to police / 
RTO / local authorities. 
• 
Potholes: detect damaged road surfaces, classify severity, track before and after state, and geo-tag the 
report. 
• 
Stray animals: detect dogs, cows, goats, pigs, and similar road hazards; classify behavior and risk. 
• 
Accidents: detect abnormal motion, collisions, sudden stops, hit-and-run behavior, and traffic disruption. 
• 
Authority reporting: create a structured incident record, send an email alert, and show the event in a live 
dashboard. 
Primary output: A working web-based product with AI detection, incident storage, live dashboard, and 
authority reporting. 
2. Non-Negotiable Constraints 
• 
Use only free or open-source tools. 
• 
Do not depend on paid APIs, credits, or billing-required services. 
• 
Keep the stack simple enough for hackathon implementation. 
• 
Prefer stable and widely used libraries over experimental or overly complex options. 
• 
Keep the architecture deterministic so a fresh AI assistant does not hallucinate extra tools or unsupported 
steps.

incident state is updated, and the dashboard plus email channel are refreshed.

6. Hazard-Specific Technical Workflows 
6.1 Pothole Detection and Lifecycle Tracking 
• 
Detect pothole bounding boxes from the video feed. 
• 
Compute pothole severity using bounding-box area, road position, and repeated detection frequency. 
• 
Store the first confirmed detection as the before state. 
• 
On later detections, match the new detection with the existing pothole using camera ID and spatial 
distance. 
• 
Update the after image and compare area changes to determine whether the pothole is growing, 
unchanged, or reduced. 
• 
If the pothole is not detected for a configured duration, mark the incident as resolved, but keep manual 
override available in the authority dashboard.

6.2 Stray Animal Detection and Behavior Risk 
• 
Detect the relevant animal types for the Indian road context: dog, cow, goat, pig, and similar stray hazards. 
• 
Classify behavior into standing, sitting, crossing, or running. 
• 
Track motion across frames to determine whether the animal is approaching the road zone. 
• 
Assign a risk score based on behavior and position within the frame. 
• 
Use the road-zone rule: if an animal enters the road zone or its trajectory intersects the road boundary, 
create an alert. 
• 
Record the hotspot location so repeated animal appearances at the same camera can be analyzed later.

6.3 Accident Detection and Severity Analysis 
• 
Use multi-frame motion analysis across a short window such as the previous 3 to 5 frames plus the 
current frame. 
• 
Track vehicles and detect abnormal patterns such as overlap, collision, and sudden stop. 
• 
Estimate severity using speed, number of vehicles, vehicle type, and human presence near the incident. 
• 
Flag possible hit-and-run behavior when an involved vehicle leaves the scene immediately after the 
collision. 
• 
Estimate traffic disruption using vehicle density before and after the accident.

9. Dashboard Requirements 
Authority dashboard: Live incident feed, filters by hazard type and severity, incident detail page, before/after 
images, map view, charts, acknowledge / resolve actions, and email history. 
Public or demo dashboard: Optional read-only view showing current hazards on a map or table for 
presentation. 
Analytics: Counts by hazard type, severity distribution, hotspot map, and recent activity timeline. 
• 
Use Leaflet with OpenStreetMap for map display. 
• 
Use Recharts for charts and summary cards. 
• 
Use polling every few seconds or Server-Sent Events for live updates. 
• 
Use a clean table layout on desktop and stacked cards on mobile. 
10. Email and Authority Reporting

• 
Attach the snapshot image or include a secure image link. 
• 
Include camera location, timestamp, hazard type, severity, and confidence score. 
• 
Use a single consistent email template for potholes, animals, and accidents. 
•

11. Recommended Build Order 
1. Create the Next.js web app, set up Tailwind CSS, shadcn/ui, Prisma, and the dashboard layout. 
2. Create the Neon PostgreSQL database and run the initial schema for cameras, incidents, snapshots, and 
history. 
3. Build the authority dashboard pages and CRUD routes first so you can see live data immediately. 
4. Build the Python FastAPI service and connect it to the video source. 
5. Integrate YOLOv8 detection for potholes, animals, vehicles, and people. 
6. Add the pothole severity engine, animal risk engine, and accident analysis engine. 
7. Add the matching logic for before / after state and resolution detection. 
8. Connect the AI service to the Next.js API route that stores incidents in Neon. 
9. Add Nodemailer SMTP email alerts and dashboard notifications. 
10. Finish with analytics, filters, maps, and presentation polish.

14. Added Enhancement: Night-Time Detection Capability 
To ensure full coverage of system requirements and enhance real-world reliability, night-time detection 
capability is integrated into the AI pipeline without altering existing architecture. 
 
Technical Enhancements: 
 
- Low-light preprocessing using histogram equalization and noise reduction. 
- Optional grayscale and contrast enhancement before YOLO inference. 
- Model fine-tuning using low-light and night-time datasets (recommended: augmentations). 
- Adaptive confidence thresholds for night conditions. 
 
Where It Applies: 
 
- Pothole Detection: Improved visibility of road surface defects under poor lighting. 
- Animal Detection: Better tracking and recognition of animals during night hours. 
- Accident Detection: Enhanced motion tracking and collision detection in low visibility. 
 
Impact: 
 
- Ensures 24/7 system reliability. 
- Reduces false negatives in night scenarios. 
- Aligns system with real-world deployment conditions.
