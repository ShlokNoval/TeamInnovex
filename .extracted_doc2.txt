2. COMPONENT DETAILS 
2.1 TESTING DASHBOARD (Frontend 1) 
Technology Stack: 
● React.js / Next.js 
● WebSocket (Socket.io client) 
● HTML5 Canvas API 
● Tailwind CSS / Material-UI 
Purpose: 
Demo interface where users can upload videos/images and see AI predictions in real-time 
Features: 
A. Upload Interface 
● Video Upload: Accepts .mp4, .avi, .mov files 
● Image Upload: Accepts .jpg, .png files 
● Live Camera Feed (Optional): For live demo 
B. Real-Time Display Panel 
● Video Player: Shows uploaded video playing 
● Annotation Overlay: Displays bounding boxes, labels, severity scores 
● Frame Info: Current timestamp, FPS counter 
C. Detection Sidebar 
● Live Detection Feed: Scrolling list of detections as they happen 
○ Pothole detections (with severity badge) 
○ Animal detections (with behavior status) 
○ Accident alerts (with severity score) 
● Count Metrics: Total detections by type 
● Filter Options: Show only high-severity alerts 
D. Upload Flow 
User selects file 
    ↓ 
File loaded in browser 
    ↓ 
Video plays in <video> element 
    ↓ 
Canvas extracts frames (every 3rd-5th frame) 
    ↓ 
Frame converted to base64 
    ↓ 
Sent to backend via WebSocket 
    ↓ 
Receives annotated frame back 
    ↓ 
Displays on screen 
Data Flow (Testing Dashboard): 
● Outgoing: Video frames (base64 JPEG, 50-80KB each) 
● Incoming: Annotated frames (base64 JPEG with boxes drawn) + Detection JSON

Database Schema: 
COLLECTIONS / TABLES: 
 
1. detections 
   ├─ id (UUID) 
   ├─ type (pothole/animal/accident) 
   ├─ severity (low/medium/high) 
   ├─ confidence (0-1) 
   ├─ bbox [x, y, w, h] 
   ├─ timestamp (ISO datetime) 
   ├─ camera_id (string) 
   ├─ metadata (JSON - specific to type) 
   └─ image_thumbnail (base64 or URL) 
 
2. alerts 
   ├─ id (UUID) 
   ├─ detection_id (foreign key) 
   ├─ severity_score (0-100) 
   ├─ alert_level (medium/high/critical) 
   ├─ status (new/acknowledged/resolved) 
   ├─ timestamp 
   └─ location_data (lat, long, camera_id) 
 
3. analytics_cache 
   ├─ metric_name (string) 
   ├─ value (number/JSON) 
   ├─ time_bucket (hourly/daily) 
   └─ last_updated 
API Endpoints: 
WebSocket: 
  /ws 
    - Bidirectional real-time communication 
    - Events: 
      • process_frame (receive from Testing Dashboard) 
      • annotated_frame (send to Testing Dashboard) 
      • new_alert (broadcast to Admin Panel) 
 
REST API: 
  POST /api/upload 
    - Alternative to WebSocket for single image 
   
  GET /api/alerts 
    - Query parameters: severity, type, start_date, end_date 
    - Returns: List of alerts 
   
  GET /api/analytics/summary 
    - Returns: Overall statistics 
    - Response: { 
        total_detections, 
        by_type {pothole, animal, accident}, 
        by_severity {low, medium, high}, 
        today_count, 
        hourly_breakdown 
      } 
   
  GET /api/analytics/heatmap 
    - Returns: Location-based frequency data 
   
  GET /api/detections/:id 
    - Get specific detection details 
   
  PATCH /api/alerts/:id 
    - Update alert status (acknowledge/resolve)

4. QUICK REFERENCE Q&A 
Q1: Does it send whole video or single frames? 
A: Single frames, one at a time. Each frame is ~80KB. 
Q2: Does backend return video or data? 
A: Both - An annotated image (with boxes already drawn) + JSON detection data. 
Q3: Does it send each frame or collectively? 
A: Each frame individually in real-time loop. 
Q4: Are boxes drawn in backend or frontend? 
A: Backend draws boxes using OpenCV (recommended). Frontend just displays the image. 
Q5: How fast is it? 
A: 100-200ms per frame = 5-10 FPS output (smooth for demo). 
Q6: Why not use Spring Boot? 
A: Python FastAPI is faster to build, natively supports ML libraries, and handles WebSockets 
easily. 
Q7: Where is the database? 
A: Connected to FastAPI backend. Stores all detections, alerts, and analytics. 
Q8: Can we demo without internet? 
A: Yes - Run everything locally (localhost). No external dependencies needed. 
Q9: What if models are slow? 
A: Skip frames (process every 3rd-5th frame), resize images to 640x480, use YOLOv8-
nano. 
Q10: How to showcase all features in demo? 
A: Prepare 3 test videos: 
● Video 1: Multiple potholes (shows severity classification) 
● Video 2: Stray animals crossing (shows behavior detection) 
● Video 3: Near-miss accident (shows accident detection + severity scoring)

  /ws 
    - Bidirectional real-time communication

  POST /api/upload 
    - Alternative to WebSocket for single image 
   
  GET /api/alerts 
    - Query parameters: severity, type, start_date, end_date 
    - Returns: List of alerts 
   
  GET /api/analytics/summary 
    - Returns: Overall statistics 
    - Response: { 
        total_detections, 
        by_type {pothole, animal, accident}, 
        by_severity {low, medium, high}, 
        today_count, 
        hourly_breakdown 
      } 
   
  GET /api/analytics/heatmap 
    - Returns: Location-based frequency data 
   
  GET /api/detections/:id 
    - Get specific detection details 
   
  PATCH /api/alerts/:id 
    - Update alert status (acknowledge/resolve)

4. QUICK REFERENCE Q&A 
Q1: Does it send whole video or single frames? 
A: Single frames, one at a time. Each frame is ~80KB. 
Q2: Does backend return video or data? 
A: Both - An annotated image (with boxes already drawn) + JSON detection data. 
Q3: Does it send each frame or collectively? 
A: Each frame individually in real-time loop. 
Q4: Are boxes drawn in backend or frontend? 
A: Backend draws boxes using OpenCV (recommended). Frontend just displays the image. 
Q5: How fast is it? 
A: 100-200ms per frame = 5-10 FPS output (smooth for demo). 
Q6: Why not use Spring Boot? 
A: Python FastAPI is faster to build, natively supports ML libraries, and handles WebSockets 
easily. 
Q7: Where is the database? 
A: Connected to FastAPI backend. Stores all detections, alerts, and analytics. 
Q8: Can we demo without internet? 
A: Yes - Run everything locally (localhost). No external dependencies needed. 
Q9: What if models are slow? 
A: Skip frames (process every 3rd-5th frame), resize images to 640x480, use YOLOv8-
nano. 
Q10: How to showcase all features in demo? 
A: Prepare 3 test videos: 
● Video 1: Multiple potholes (shows severity classification) 
● Video 2: Stray animals crossing (shows behavior detection) 
● Video 3: Near-miss accident (shows accident detection + severity scoring)
